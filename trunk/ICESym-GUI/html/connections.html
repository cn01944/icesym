Connection Between Objects

Perhaps the most important operation to do is the connection between components. For an operational engine model the first requeriment is the relation between its parts in a natural way. This natural way determines what objects can be connected with what others.

Moreover exists others restrictions that limit the connections possibilities. For understand it, is necessary understand how indicates the direction in the connection. Actually the form is the selection order. When the user want to connect the objects, first must select the left object (left, not means left in the screen), and second right click in the right object, in the simulation this means that the exhaust in the first object is intake in the second object. But, be careful, that some object only can be connected with only one object. This system rules borns in the real engines, and can be resumed in the next table:

Object From Object To Number Intakes Number Exhaust
Tube All 1 1
Junction Tube N N
Atmosphere Tube 1 or 0 1 or 0
Cylinder Exhaust Valve N N
Intake Valve Tube (in intake) - Cylinder (in exhaust) 1 1
Exhaust Valve Cylinder (in intake) - Tube (in exhaust) 1 1
Tank Tube N N



Example: Connecting Objects

In this example we shall to explain how connect the next simulation, composed for: Atmosphere-Tube-Atmosphere (for example, if we need verify the behavior of the flux in the tube when exists differents conditions in the intake atmosphere and the exhaust atmosphere).
  • 1 - Create the three objects: two atmospheres and one tube (see the first image)


  • 2 - Select the first Atmosphere and do right click over the Tube. This action will connect the objects.
  • 3 - Select the Tube and do right click over the second Atmosphere. Now we have the connections required.


  • 4 - For remove connections only select the line that represents the connection, and press "supr" key.
